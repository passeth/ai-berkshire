# AI Berkshire - AI 시대의 가치 투자 연구 프레임워크

> "Price is what you pay, value is what you get." — Warren Buffett
>
> AI로 투자 연구의 깊이와 효율성을 재정의합니다.

**AI Berkshire**는 [Claude Code](https://claude.ai/code) 기반의 투자 연구 Skill 모음입니다. 워렌 버핏, 찰리 멍거, 단용평, 리루 등 가치 투자 4대 거장의 방법론을 체계화·구조화하고, AI Agent를 통해 전문가 수준의 투자 연구를 수행합니다.

한 사람 + Claude = 하나의 투 research 팀.

## 왜 AI에게 그냥 물어보면 안 되나?

물론 Claude에게 "핀둬둬 사도 될까?"라고 직접 물어볼 수 있습니다. 그러면 "한편으로는... 다른 한편으로는..." 식의 균형 잡힌 분석이 나오고, 마지막에 "투자는 위험이 따르니 스스로 판단하세요"로 끝납니다.

**겉보기에는 맞아 보이지만, 실제 의사결정에 쓸 수 없습니다.**

AI Berkshire가 해결하는 것은 "분석할 수 있는가"가 아니라 **분석의 품질과 의사결정의 규율** 문제입니다. 핵심 차이점은 다음과 같습니다:

### 1. 결론을 강제로 내린다 — 회피하지 않는다

직접 AI에게 물으면 양쪽을 다 만족시키는 "분석"이 나옵니다. AI Berkshire는 **통과/불통과/회색지대**를 강제로 출력하고, 구체적인 가격 구간과 단계별 제안을 함께 제공합니다.

> 일반 AI 답변: *"핀둬둬는 성장 잠재력이 있지만 경쟁 압력도 있어서 투자자가 잘 저울질해야 합니다..."*
>
> AI Berkshire 출력:

> | 전략   | 제안                     | 가격 구간 |
> |--------|--------------------------|-----------|
> | 공격형 | 현재 가격대에서 20% 매수 | $95-105   |
> | 안정형 | 자사주 매입 정책 명확화 후 매수 | $85-95 |
> | 보수형 | 10년 확실성 기준 미충족, 관망 | — |

> **거울 테스트**: 5문장으로 설명이 안 되면 사지 않는다. 예외 없음.

### 2. 4대 거장 관점의 대립 — 단일 분석이 아니다

"버핏 방식으로 분석해줘"가 아닙니다. 네 가지 관점은 **실제 모순과 긴장**을 만들어냅니다.

핀둬둬 예시:
- **단용평** (비즈니스 모델): 좋은 사업, C2M 모델 복제 어려움 → 3.7/5
- **버핏** (재무·밸류에이션): 현금 차감 PE 6.3배, 현금 인쇄기 → 4.4/5
- **멍거** (역발상): 해자(경쟁우위)가 생각보다 얕음, 도우인 3년 만에 4조 GMV → 3.5/5
- **리루** (장기 확실성): 경영진 문화에 문제 소지, 10년 후 불확실 → 2.0/5

**버핏은 "진짜 싸다"고 하고, 리루는 "불확실하면 사지 마"라고 합니다.** — 이런 충돌이야말로 투자 의사결정의 실제 모습입니다. 단일 프롬프트로는 이런 다중 관점 대립을 만들 수 없으며, 이것이 바로 맹점을 피하는 핵심입니다.

### 3. 구조화된 편향 방지 메커니즘

AI의 가장 큰 위험은 틀린 답을 주는 것이 아니라 **겉으로는 맞아 보이지만 검증되지 않는** 답을 주는 것입니다. AI Berkshire에는 여러 층의 "속임수 방지" 장치가 내장되어 있습니다:

| 메커니즘             | 해결하는 문제             | 예시                              |
|----------------------|---------------------------|-----------------------------------|
| **정보 풍부도 등급(A/B/C)** | "자료 많음 = 확실성 높음" 착각 방지 | 버블마트 B등급: 데이터 제한적, 추정 지표에 신뢰도 표기 |
| **멍거식 역발상 검증** | 실패 시나리오 강제 사고   | "핀둬둬가 망할 상황은?" → 5대 시나리오 및 확률 나열 |
| **신속 거부 목록**   | 8가지 레드라인 즉시 거부  | 경영진 신뢰성 문제 → 밸류에이션과 무관하게 즉시 거부 |
| **반(反)컨센서스 점검** | 시장 생각과 같아지는 것 방지 | "똑똑한 사람들이 왜 공매도하나?" → 간과된 리스크 발견 |
| **공백 원칙**        | "모른다"고 말할 용기      | 데이터 부족 시 "회색지대" 명시, 추측으로 확실성 위장 금지 |

### 4. 금융 데이터의 정확성

LLM의 암산은 신뢰할 수 없습니다. PE를 소수점 하나 틀리거나, 시가총액 단위를 홍콩달러와 위안으로 혼동하면 잘못된 투자 결정으로 이어질 수 있습니다.

**실제 사례**: 텐센트 분석 시, 출처에 따라 시가총액 데이터가 "홍콩달러 억"과 "위안 억" 두 가지 단위로 나왔습니다. AI Berkshire의 처리 방식:

```bash
# 시가총액 수작업 검증: 주가 × 발행 주식수 vs 보고서 데이터 비교
python3 tools/financial_rigor.py verify-market-cap \
  --price 510 --shares 9.11e9 --reported 4.65e12 --currency HKD
# ✅ 검증 통과, 오차 0.08%만
```

모든 계산은 Python `decimal.Decimal`(정확한 십진수)을 사용하며 `float`를 쓰지 않습니다. 핵심 데이터는 최소 2개 독립 출처로 교차 검증합니다.

### 5. 재현 가능한 연구 프로세스

AI에게 직접 물으면 매번 출력 형식·깊이·범위가 다릅니다. 오늘 텐센트 분석에는 해자 점수가 있는데, 내일 메이퇀 분석에서는 그걸 잊어버릴 수 있습니다.

AI Berkshire는 **같은 입력 → 구조와 깊이가 일관된 출력**을 보장합니다. 이는 다음을 의미합니다:
- 7개 회사 횡단 비교 시, 평가 기준이 완전히 동일
- 같은 회사를 반년 후 재분석해 직접 변화 비교 가능
- 팀원 간 연구 결과 정렬 가능

> 실제 출력 — 7개 회사를 동일한 표준 Checklist로 선별:
>
> | 회사     | 통과?   | 능력권 | 좋은 사업 | 해자   | 경영진 | 안전마진 | 종합 |
> |----------|---------|--------|-----------|--------|--------|----------|------|
> | 마오타이 | ✅ 통과 | ★★★★★ | ★★★★★     | ★★★★★ | ★★★☆☆ | ★★★★☆    | 4.7  |
> | 텐센트   | ✅ 통과 | ★★★★☆ | ★★★★★     | ★★★★★ | ★★★★★ | ★★★★☆    | 4.7  |
> | 엔비디아 | ✅ 조건부 | ★★★★☆ | ★★★★★   | ★★★★★ | ★★★★★ | ★★★☆☆    | 4.3  |
> | 메이퇀   | ✅ 조건부 | ★★★★☆ | ★★★★☆   | ★★★★☆ | ★★★★☆ | ★★★★☆    | 4.0  |
> | 콰이서우 | ✅ 조건부 | ★★★☆☆ | ★★★★☆   | ★★★★☆ | ★★★★☆ | ★★★★★    | 4.0  |
> | 핀둬둬   | ❓ 회색  | ★★★★☆ | ★★★★☆    | ★★★☆☆ | ★★★☆☆ | ★★★★★    | 3.8  |
> | 버블마트 | ❓ 회색  | ★★★☆☆ | ★★★★☆    | ★★★★☆ | ★★★★★ | ★★★☆☆    | 3.7  |

### 6. 다중 Agent 병렬 = 연구 깊이 배가

`/investment-team`은 4개의 독립 Agent를 **동시에** 작동시켜 한 회사를 연구합니다. 각 Agent는 각각 네트워크 검색, 데이터 교차 검증, 독립 결론 도출을 수행합니다. 하나의 프롬프트를 네 조각으로 쪼개는 게 아니라, 4명의 "애널리스트"가 각각 완전한 연구를 수행하고, Team Lead가 종합하는 구조입니다.

한 사람이 AI에게 직접 물으면 컨텍스트 창은 하나입니다. 4개 Agent 병렬 실행은 검색량 4배, 정보원 4배, 독립 관점 4개를 의미합니다.

```
┌─────────────────────────────────────────────┐
│              Team Lead (당신)                │
│         총괄 조율 · 종합 판단                 │
├──────┬──────┬──────────┬───────────┤
│ Agent 1    │ Agent 2    │ Agent 3        │ Agent 4         │
│ 비즈니스 모델│ 재무·밸류에이션│ 업계 경쟁       │ 리스크·경영진     │
│ 단용평 관점  │ 버핏 관점   │ 멍거 관점       │ 리루 관점         │
└──────┴──────┴──────────┴───────────┘
        ↓ 병렬 연구, 실시간 진행 보고 ↓
              최종 종합 보고서
```

## 빠른 시작

### 1. Claude Code 설치

```bash
npm install -g @anthropic-ai/claude-code
```

### 2. Skills 설치

`skills/` 디렉토리의 `.md` 파일들을 Claude Code commands 디렉토리로 복사합니다:

```bash
# 레포 클론
git clone https://github.com/passeth/ai-berkshire.git

# skills를 Claude Code 전역 commands 디렉토리로 복사
cp ai-berkshire/skills/*.md ~/.claude/commands/
```

### 3. 사용

Claude Code에서 직접 호출:

```bash
# 심층 연구
```

## 각 Skill 상세 소개

### 1. `/investment-research` — 4대 거장 종합 분석

가장 포괄적인 단일 회사 심층 연구 프레임워크. 7개 모듈 순서로 연구를 실행합니다:

```
데이터 수집 → 비즈니스 본질(단용평) → 해자(버핏) → 역발상 사고(멍거)
    → 경영진 평가(단용평+버핏) → 문명 트렌드(리루) → 밸류에이션 및 안전마진
```

**핵심 특징**:
- AI 연구 편향 자각 메커니즘 (A/B/C급 정보 풍부도 등급)
- 핵심 데이터 다중 출처 교차 검증 (시가총액 수작업 검산, 최소 2개 독립 출처)
- 4대 거장의 "추가 질문"이 보고서 전반에 걸쳐 등장
- 3시나리오 밸류에이션 (낙관/중립/비관) + 역DCF

**출력 예시 발췌**:

> #### 종합 의사결정 메모
>
> | 차원 | 결론 | 신뢰도 |
> |------|------|--------|
> | 비즈니스 품질 (단용평) | 극우수: 플랫폼형 사업, 양면 네트워크 효과, 한계비용 0에 수렴 | ★★★★★ |
> | 해자 (버핏) | 넓고 더 넓어지는 중: 네트워크 효과+전환 비용+규모 효과 3중 중첩 | ★★★★☆ |
> | 경영진 (단용평+버핏) | 우수: 창업자가 키를 쥐고 있으며 자본 배분 규율이 강함 | ★★★★☆ |
> | 최대 리스크 (멍거) | 규제 정책 불확실성, 신사업 적자가 전체 이익을 압박 | ★★★☆☆ |
> | 문명 트렌드 (리루) | 디지털 소비 트렌드에 순응하나 "문명급 패러다임 전환"은 아님 | ★★★★☆ |
> | 밸류에이션 (버핏+단용평) | 현재 PE 18배, 역사적 중간값보다 낮아 일정 수준 안전마진 존재 | ★★★★☆ |
>
> **단용평**: "이 사업의 본질은 소비자와 상인을 연결하는 것이며, 효율성 향상으로 돈을 버는 것입니다. 좋은 사업의 표지는: 이용자가 많을수록 상인이 많아지고, 상인이 많을수록 이용자가 많아진다는 것입니다. 한번 플라이휠이 돌기 시작하면 멈추기 어렵습니다."
>
> **멍거**: "반대로 생각해보자 — 만약 이 회사가 내일 사라진다면, 이용자와 상인은 어떻게 될까? 답이 '빠르게 대체재를 찾을 수 있다'라면 해자가 충분히 깊지 않은 것입니다. 답이 '생활이 매우 불편해질 것이다'라면 주목할 만합니다."

### 2. `/investment-team` — 다중 Agent 투 research 팀

4개의 AI Agent를 병렬로 가동해 실제 투 research 팀 협업을 시뮬레이션합니다. 각 Agent는 독립적으로 검색·분석·평점을 매기고, 마지막에 Team Lead가 종합 판단합니다.

**출력 예시 발췌**:

> #### 한 줄 결론
> 메이퇀은 중국 로컬 생활 서비스의 절대적 선두주자로, 다중 네트워크 효과 해자를 보유하고 있으며 현재 밸류에이션이 역사적으로 낮은 수준에 있어 장기 투자 가치가 뚜렷합니다. 저점 매수를 권장합니다.
>
> #### 4차원 평가 종합표
>
> | 차원 | 프레임워크 | 평점 | 핵심 판단 |
> |------|------------|------|-----------|
> | 비즈니스 모델 & 해자 | 단용평 | ★★★★☆ | 양면 네트워크 효과 강력, 배달+매장 시너지 플라이휠 형성 |
> | 재무 & 밸류에이션 | 버핏 | ★★★★☆ | 핵심 사업 이익률 지속 개선, 밸류에이션 역사적 저점 |
> | 업계 & 경쟁 | 멍거 | ★★★☆☆ | 도우인이 매장 사업 침공, 경쟁 구도 악화 리스크 |
> | 리스크 & 경영진 | 리루 | ★★★★☆ | 왕싱 전략적 안목 우수하나 신사업 자금 소모 주의 필요 |
>
> **종합 평점: 3.8 / 5**
>
> #### 투자 제안
>
> | 전략 | 제안 | 가격 구간(홍콩달러) |
> |------|------|-------------------|
> | 공격형 | 현재 가격대에서 30% 매수 | 120-140 |
> | 안정형 | 100-110으로 조정 시 매수 | 100-120 |
> | 보수형 | 분기 실적으로 이익률 추세 확인 후 진입 | <100 |

### 3. `/investment-checklist` — 버핏 매수 전 Checklist

6단계 빠른 선별로 10분 안에 한 회사가 깊이 연구할 가치가 있는지 결정합니다:

```
1단계: 능력권 (내가 이해할 수 있는가?)
    ↓ 통과
2단계: 좋은 사업 (경제적 특성은 어떠한가?)
    ↓ 통과
3단계: 해자 (경쟁우위가 깊은가?)
    ↓ 통과
4단계: 경영진 (신뢰할 수 있는가?)
    ↓ 통과
5단계: 안전마진 (가격이 싼가?)
    ↓ 통과
6단계: 의사결정 규율 (이성인가 FOMO인가?)
    ↓ 통과
   ✅ 거울 테스트
```

**다중 회사 비교 지원** — 여러 종목을 한 번에 선별:

```
/investment-checklist 텐센트, 알리바바, 메이퇀, 핀둬둬
```

**출력 예시 발췌**:

> #### 거울 테스트
>
> "나는 380홍콩달러에 텐센트를 매수한다. 이유는 다음과 같다:
>
> 1. 이 사업의 본질은 **소셜 네트워크 + 디지털 콘텐츠 플랫폼**이며, 나는 이를 이해한다.
> 2. 해자는 **12억 사용자의 소셜 관계망**이며, 더 넓어지고 있다.
> 3. 경영진 **Pony Ma는 저자세·실무적이며 자본 배분이 우수**해 신뢰할 수 있다.
> 4. 현재 가격은 내재 가치의 **8할** 수준으로 일정 안전마진이 있다.
> 5. 설령 틀리더라도 하락 리스크는 통제 가능하다. **장부상 순현금 2,000억 초과 + 게임 현금흐름 강력**하기 때문이다."
>
> ✅ 거울 테스트 통과
>
> **5문장으로 설명이 안 되면 사지 않는다. 예외 없다.**

### 4. `/industry-research` — 산업 가치사슬 전경 스캔

하나의 투자 테마에서 출발해 산업 가치사슬 전체를 연구합니다:

```
투자 논리 체인 구축 → 산업 가치사슬 전경도 → 글로벌 상장사 스캔
    → 각 단계 선두 기업 4대 거장 분석 → 투자 포트폴리오 배분 제안
```

**출력 예시 발췌**:

> #### 투자 논리 체인: 원전
>
> 기반 트렌드: AI 데이터센터 전력 수요 폭발 + 탄소중립 목표
> → 결과: 안정적 청정 기저부하 전원 수요 급증
> → 창출: 원전 재가동/신설/SMR의 확실한 수요
> → 수혜: 우라늄 광산 → 연료 가공 → 설비 제조 → 운영사
>
> #### 추천 포트폴리오
>
> | 계층 | 비중 | 종목 | 단계 | 핵심 논리 |
> |------|------|------|------|-----------|
> | 핵심 | 50% | 중국광핵(CGN), Cameco | 운영+우라늄 | 확실성 최고 |
> | 위성 | 30% | 중국원전, 동방전기 | 운영+설비 | 국산 대체 수혜 |
> | 옵션 | 15% | NuScale, Nano Nuclear | SMR | 고위험 고탄력 |
> | ETF | 대안 | URA, URNM | 전체 | 간편 솔루션 |

### 5. `/industry-funnel` — 업계 깔때기 선별

하나의 산업/방향에서 출발해 **전시장 → ≤10사 → 3사** 단계별 정밀 선별:

```
전시장 스캔 (활성도 + 상승률 + 시총 상위 30 합집합, 30-60사)
    ↓ 가치 투자 5대 하드 지표
1차 선별 ≤ 10사
    ↓ 정밀 분석 (사당 300-500자)
정밀 분석 ≤ 10사
    ↓ 최종 선별 (조합 상호보완성 기준, 점수 상위 3이 아님)
4대 거장 심층 분석 3사 (사당 800-1200자)
    ↓
추천 조합 (핵심 / 위성 / 옵션) + 운영 시그널
```

**핵심 특징**:
- 각 단계마다 명확한 유지/제외 기준, 탈락 종목에 탈락 사유 명시 (블랙박스 아님)
- 최종 3사는 "조합 상호보완성"으로 선정 (고확실성 + 중간 탄력 + 고탄력), 점수 상위 3으로 정렬하지 않음
- "미래 IPO 후보"를 강제로 나열해 1차 시장 핵심 플레이어 누락 방지
- AI 편향 자각 메커니즘: 리더 선호 / 영어 선호 / 스토리 선호 / 상장 선호 대응

**`/industry-research`와의 차이**:
- `industry-research`는 가치사슬 구조와 전경에 중점 (단계별 슬라이스)
- `industry-funnel`은 개별 종목 선별 깔때기에 중점 (전시장 → 3사)

**실측: AI 산업 4개 하위 섹터 병렬 (2026-05-09)**:

| 하위 섹터 | 최종 3사 | 핵심 비중 추천 |
|-----------|----------|----------------|
| AI 컴퓨트 | TSMC / NVIDIA / SK하이닉스 | TSMC ★★★★★ |
| AI 모델 | Alphabet / Meta / 알리바바 | Alphabet ★★★★★ |
| AI 애플리케이션 | 마이크로소프트 / Adobe / AppLovin | 마이크로소프트 + Adobe ★★★★ |
| AI 인프라 전력 | Eaton / 특변전공 / Talen Energy | Eaton + 특변전공 ★★★★ |

**주요 발견**: AI 애플리케이션 층의 최대 승자는 AI 네이티브 회사가 아니라 채널+데이터+워크플로우 임베디드 성숙 대형주 — 이는 1995-2000 인터넷 버블 당시 "삽을 파는 사람"이 이겼던 역사 법칙(아마존·애플 승, Pets.com 패)과 일치합니다.

### 6. `/private-company-research` — 비상장사 심층 연구

정보가 희박한 비상장사를 위한 "탐정식" 연구 프레임워크:

**핵심 차별화**:
- **재무 데이터 퍼즐 맞추기**: 상장 신청서, 모회사 재무제표, 자금 조달 뉴스, 업계 데이터 다중 출처로 조합
- **신뢰도 표기**: 각 데이터 포인트에 🟢고 / 🟡중 / 🔴저 신뢰도 표기
- **다중 방법 밸류에이션 교차**: 자금 조달 밸류에이션법 + 유사 회사법 + DCF + 종국 역산법
- **엑시트 경로 분석**: IPO/인수합병/2차 양도 전 경로 평가

**출력 예시 발췌**:

> #### 회사 프로필 요약: SpaceX
>
> | 항목 | 내용 |
> |------|------|
> | 최신 밸류에이션 | ~$350B (2025년 2차 시장) 🟡 |
> | 추정 매출 | ~$130억 (2024년) 🟡 |
> | Starlink 사용자 | 400만+ (2024년 말) 🟢 |
> | 발사 횟수 | 연 100회+ (2024년) 🟢 |
>
> #### 밸류에이션 판단
>
> | 방법 | 밸류에이션 구간 | 설명 |
> |------|----------------|------|
> | 최근 자금 조달 | $350B | 2차 시장 호가, 유동성 프리미엄 포함 |
> | 유사 회사법 | $200-280B | 통신+항공우주+국방 벤치마크 |
> | DCF(중립) | $250-350B | Starlink 2027년 $300억 매출 가정 |
> | 종국 역산 | $400-600B | 스타링크가 글로벌 통신 인프라가 된다는 가정 |
>
> **종합 합리적 밸류에이션 구간: $250B - $400B**

### 7. `/news-pulse` — 주가 급변동 뉴스 귀인

"주가가 크게 오르거나 떨어질 때 무슨 일이 있었는지 빠르게 파악"하기 위해 설계된 정보 대응 Skill입니다. **심층 투 research가 아니라 10-15분 빠른 귀인** — 보유 종목 변동 시 작은 에세이 불안이나 맹목적 손절에 빠지는 것을 방지합니다.

**핵심 차별화**:
- **4차원 병렬 정찰**: 회사 이벤트 / 규제 정책 / 업계 경쟁사 / 시장 심리 (매도 측 + 인플루언서 + 남향 자금)
- **귀인이 목록화보다 우선**: 모든 뉴스를 나열하는 것이 아니라 "어느 이벤트가 이번 주가 변동에 상응하는가"를 판단
- **강제 성격 판단**: 가치 이벤트 / 심리 변동 / **진짜 원인 불명** / 혼합 — 여기서 "진짜 원인 불명"이 가장 가치 있는 출력 (내부자 선행 가능성)
- **명확한 행동 제안**: 심층 연구 트리거 여부, 논리 재검토 필요 여부, 단순 관찰 등

**다른 Skill과의 차이**:

| 상황 | 무엇을 사용할까 |
|------|-----------------|
| 완전 투 research (시간 단위) | `/investment-team` 또는 `/investment-research` |
| 재무제표 정독 | `/earnings-review` |
| 장기 논리 추적 | `/thesis-tracker` |
| **주가 급변동 10분 귀인** | **`/news-pulse`** |

**출력 예시 발췌** (텐센트 4/17-5/01 실측, 14일 -10.47%):

> #### 한 줄 귀인

## 설계 철학

### 4대 거장 방법론 융합

```
              ┌──────────────────┐
              │    단용평         │
              │  "옳은 사업"      │
              │  비즈니스 모델 본질  │
              └────────┬─────────┘
                       │
    ┌──────────────────┼──────────────────┐
    │                  │                  │
    ▼                  ▼                  ▼
┌────────┐     ┌──────────┐      ┌────────┐
│ 버핏   │     │   멍거    │      │  리루   │
│ 해자   │     │ 역발상    │      │ 문명 트렌드│
│ 안전마진│     │ 리스크 목록│      │ 패러다임 전환│
│ 경영진 │     │ 편향 자가 점검│      │ 산업 가치  │
└────────┘     └──────────┘      └────────┘
```

4대 거장은 단순한 역할 분담이 아니라 **서로 도전**하도록 설계되었습니다:
- 단용평이 "좋은 사업"이라고 하면 멍거는 "어떻게 망할까?"라고 묻습니다.
- 버핏이 "충분히 싸다"고 하면 리루는 "10년 후에도 존재할까?"라고 묻습니다.
- 당신이 얻는 것은 네 장의 보고서를 이어붙인 것이 아니라, 네 가지 사고 방식의 충돌입니다.

### 금융 엄밀성 도구 (`tools/financial_rigor.py`)

| 기능 | 명령 | 해결하는 문제 |
|------|------|---------------|
| **시가총액 검산** | `verify-market-cap` | 주가×발행주식수 정확 계산, 단위 오류 탐지 |
| **밸류에이션 검산** | `verify-valuation` | PE/PB/ROE/FCF Yield 정확 십진수 계산 |
| **다중 출처 교차 검증** | `cross-validate` | N개 출처의 동일 데이터 자동 비교, 허용 오차 초과 시 경고 |
| **3시나리오 밸류에이션** | `three-scenario` | 낙관/중립/비관 정확 계산 목표가 |
| **Benford 법칙 탐지** | `benford` | 재무 데이터 선두 숫자 분포 이상 탐지 |
| **정확 계산기** | `calc` | 임의 재무 표현식 정확 계산, LLM 암산 대체 |

**설계 원칙**: 모든 계산은 Python `decimal.Decimal`(정확한 십진수)을 사용하며 `float`(부동소수점 근사)를 사용하지 않습니다. 금융 장면에서는 `0.1 + 0.2 = 0.3`이 실패해서는 안 됩니다.
