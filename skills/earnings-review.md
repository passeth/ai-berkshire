# 재무제표 정독: 1차 자료 심층 해석

$ARGUMENTS 에 대해 재무제표 정독 분석을 수행합니다.

**입력 형식 지원**: `회사명 분기`, 예: `텐센트 2025Q4`, `PDD 2025년 연례 보고서`, `메이퇀 최신` (기본: 최근 분기)

> "나는 매도 측 리서치 보고서를 절대 보지 않고, 원본 재무제표만 읽습니다." — 리루
>
> "나는 매일 500페이지를 읽습니다. 지식은 그렇게 복리로 쌓입니다." — 버핏

## 설계 철학

대부분의 AI 투 research 도구는 2차 정보(뉴스, 리서치 요약, 데이터 사이트)에 의존합니다. 그러나 버핏과 리루의 핵심 역량은 **1차 자료 읽기** — 연례 보고서, 분기 보고서, 실적 발표회 의사록입니다.

2차 정보의 문제:
- 선별됨 — 애널리스트가 자신의 견해에 유리한 데이터만 선택적으로 제시
- 시차 존재 — 남이 소화한 후에는 알파가 이미 사라짐
- 맥락 부족 — "매출 15% 성장"이 성장 품질에 대한 경영진 논의와 분리됨

이 Skill은 1차 자료를 직접 해석하며, 버핏과 리루가 실제로 볼 내용에 집중합니다.

## 실행 흐름

### 전제 단계: 자료 가용성 등급

| 등급 | 특징 | 영향 |
|------|------|------|
| A급 | 완전한 원문 확보 (10-K/연례 보고서/실적 발표회 의사록) | 정상적으로 모든 단계 실행 |
| B급 | 부분 원문 또는 제3자 요약만 확보 | "비원본 출처" 표기, 주석 분석 비중 축소 |
| C급 | 뉴스 보도와 데이터 사이트 요약만 존재 | 핵심 재무 데이터 변화에 집중, 주석 파기 생략, "1차 자료 부족" 표기 |

### 첫 번째 단계: 1차 자료 확보

Task 도구를 사용해 여러 백그라운드 Agent를 **병렬**로 시작하여 다음 원본 자료를 확보합니다:

1. **재무제표 원문**: 회사 IR 페이지, SEC EDGAR (미국주 10-K/10-Q), 홍콩 거래소 공시 (홍콩주), 거조자신망 (A주)에서 확보
2. **실적 발표회 의사록/녹음**: Seeking Alpha, 회사 IR 페이지, 쉐볼 등에서 확보
3. **경영진 주주 서한** (연례 보고서에 있는 경우): 전체 읽기
4. **투자자 데이/애널리스트 데이 자료** (최근에 있었을 경우)

완전한 원문을 확보할 수 없는 경우, `skills/financial-data.md` 규범에 따라 표준 데이터 소스를 사용해 조합하되 (미국주: macrotrends+stockanalysis; 홍콩주: aastocks+macrotrends; A주: 동방재부+거조자신), 반드시 "비원본 재무제표, 제3자 요약 출처"라고 표기하고, 핵심 데이터 두 출처 오차 >1% 시 표시해야 합니다.

### 두 번째 단계: 핵심 재무 데이터 추출 및 검증

#### 2.1 매출 및 손익 계산서

| 지표 | 당기 | 전기 | YoY 변화 | 경영진 가이던스 | 달성 여부 |
|------|------|------|----------|------------------|-----------|

반드시 포함:
- 총 매출 및 사업부/지역별 매출 분해
- 매출총이익, 매출총이익률 변화
- 영업이익, 영업이익률 변화 (GAAP와 Non-GAAP 구분)
- 순이익 (비반복적 손익 영향 주의)
- EPS (기본 vs 희석)

#### 2.2 현금흐름표 (버핏이 가장 중시)

| 지표 | 당기 | 전기 | 변화 | 주목점 |
|------|------|------|------|--------|

반드시 포함:
- 영업 현금흐름 vs 순이익 비율 (>100% 우수, <80% 주의)
- 자본적 지출 및 구성 (유지 vs 확장)
- 잉여현금흐름 = 영업 현금흐름 - 자본적 지출
- 자사주 매입 금액, 배당 금액
- 현금 및 현금성 자산 기말 잔액

#### 2.3 재무상태표 건전성

반드시 포함:
- 현금 + 단기투자 vs 유이자부채
- 순현금/순부채 변화 추이
- 매출채권 회전일수 변화 (신용 조건 완화로 매출을 밀어내고 있는가?)
- 재고 회전일수 변화 (재고가 쌓이고 있는가?)
- 영업권 및 무형자산 비중 (감액 리스크 있는가?)

**데이터 검증**: `tools/financial_rigor.py` 를 사용해 핵심 데이터를 검증:

```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation \
  --price {주가} --eps {EPS} --bvps {주당 순자산}
```

### 세 번째 단계: 경영진 논의 및 전망 분석

#### 3.1 실적 발표회에서 강조된 내용

- 경영진이 반복해서 강조한 키워드 (성장 동력, 마진, 비용 통제 등)
- "이번 분기 하이라이트"와 "다음 분기 중점"
- 가이던스 변경 이유와 그 이면의 의미
- 애널리스트 질문 중 가장 많이 나온 주제와 경영진 답변의 솔직성

#### 3.2 관리회계적 관점 (숫자 이면)

- 매출 성장이 가격 인상에 의한 것인가, 물량 증가에 의한 것인가?
- 비용 절감이 구조적(영구적)인가, 일회성(일시적)인가?
- 신사업 손실이 투자 단계에서 예상된 것인가, 실행 미스인가?
- 회계 정책 변경(수익 인식, 상각 등)이 숫자에 어떤 영향을 주었는가?

### 네 번째 단계: 리스크와 주의점 식별

- "경영진이 솔직하게 말하지 않은" 리스크 (전화회 녹취/의사록에서 드러나는 뉘앙스)
- 분기별 변동성이 큰 항목과 그 이유
- 경쟁사 대비 악화되는 지표
- 규제/정책 관련 언급의 강도와 빈도

### 다섯 번째 단계: 종합 판단

| 항목 | 판단 | 근거 |
|------|------|------|
| 사업 펀더멘털 변화 | 개선 / 유지 / 악화 | |
| 이익 품질 | 높음 / 보통 / 낮음 | |
| 현금 창출력 | 강함 / 보통 / 약함 | |
| 경영진의 투명성 | 높음 / 보통 / 낮음 | |
| 투자 관점 함의 | | |

## 출력 요구사항

1. 원본 재무제표에서 직접 인용한 숫자와 문장을 사용 (2차 요약 금지)
2. 모든 숫자는 전기 대비 변화와 경영진 가이던스 대비 달성 여부를 함께 표시
3. 현금흐름 분석 비중을 높임 (버핏 스타일)
4. 경영진의 말과 실제 숫자 사이의 괴리를 명확히 지적
5. 최종 보고서는 `~/[회사명]-earnings-[기간].md` 에 작성
6. "한 줄 요약" + "숫자 변화표" + "경영진 논의 핵심" + "투자 시사점" 구조 유지

## 예시 (요약)

**회사**: 텐센트 (0700.HK) 2025Q4

**한 줄 요약**: 게임 매출 회복 + 광고/핀테크 안정적 성장으로 전체 이익률 개선. 클라우드 손실 축소가 핵심 긍정 요인.

**현금흐름 핵심**: 영업 현금흐름 / 순이익 = 132% (양호). 자사주 매입 150억 홍콩달러.

**주의점**: 신규 게임 라인업 집중도가 높아 2026년 상반기 출시 성과에 따라 변동성 확대 가능.

**판단**: 펀더멘털은 안정적이며, 현재 밸류에이션에서 추가 매수 매력 있음. 5% 비중 상향 고려.
