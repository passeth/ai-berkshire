# 투 research 팀: 4역할 병렬 분석 프레임워크

$ARGUMENTS에 대해 팀 방식의 투자 연구 분석을 수행합니다. Team 도구를 사용해 진짜 다중 Agent 병렬 연구 팀을 만듭니다.

## 실행 흐름

### 1단계: 팀 프레임워크 제시

다음 팀 구조를 사용자에게 제시하고 확인 후 시작:

| 역할 | 책임 | 분석 프레임워크 |
|------|------|-----------------|
| **team-lead** (당신) | 총괄 조율, 종합 판단, 최종 보고서 출력 | 4대 거장 종합 프레임워크 |
| **business-analyst** | 비즈니스 모델 & 해자 분석 | 단용평 관점 |
| **financial-analyst** | 재무제표 & 밸류에이션 분석 | 버핏 관점 |
| **industry-researcher** | 업계 구도 & 경쟁 태세 | 멍거 관점 |
| **risk-assessor** | 리스크 평가 & 경영진 판단 | 리루 관점 |

### 1단계 반: AI 연구 편향 평가

팀을 만들기 전에 해당 회사의 "AI 연구 가능성" 평가를 사용자에게 제시:

**정보 풍부도 등급** (연구 전략 결정):
| 등급 | 특징 | 연구 전략 조정 |
|------|------|----------------|
| A급 (정보 충분) | 상장 다년, 증권사 커버리지 광범위 | 팀은 **역발상 검증**과 **비컨센서스 관점**에 집중. 시장과 일치하는 "정확한 헛소리" 출력 방지 |
| B급 (정보 보통) | 상장 초기, 커버리지 제한적 | 각 Agent의 추정 데이터는 반드시 신뢰도 표기. team-lead 종합 시 "데이터 충분도" 명시 |
| C급 (정보 희박) | 비인기/신규 상장/신흥 시장 | 팀을 "제1원칙 모드"로 전환: 보고서 완전성 추구하지 말고, 비즈니스 본질의 몇 가지 핵심 질문에 집중 |

**중요 알림**: 자료가 많다고 확실성이 높은 것은 아니며, 자료가 적다고 확실성이 낮은 것도 아니다. AI가 출력할 수 있는 신뢰도 ≠ 투자의 진짜 확실성. 확실성은 사업 모델 자체에서 나오며, 자료의 양에서 나오지 않는다.

등급 결과를 각 Agent에게 알려 연구 방식에 영향을 주도록 한다.

### 2단계: 팀 생성

TeamCreate를 사용해 팀 생성:
- team_name: `{회사명}-research` (영문 소문자, 예: `meituan-research`)
- agent_type: `team-lead`

### 3단계: 4개 태스크 생성

TaskCreate를 사용해 다음 4개 태스크 생성 (각각 subject, description, activeForm 필요):

#### 태스크 1: 비즈니스 모델 분석
- subject: `{회사명} 비즈니스 모델, 해자 및 사용자 가치 분석`
- description 포함:
  1. 비즈니스 모델 본질: 핵심 사업 정의, 수익 구조 분해
  2. 플랫폼/제품 플라이휠 효과가 어떻게 작동하는가
  3. 해자 분석: 브랜드/전환 비용/네트워크 효과/규모 효과/기술 장벽 — 항목별 검증
  4. 사용자/고객 가치: 각 주체를 위해 창출한 고유 가치
  5. 사업 매트릭스와 시너지 효과
  6. 단용평 "좋은 사업" 기준 평가: 차별화, 가격 결정력, 지속 가능한 경쟁우위
  7. 최신 재무제표, 업계 보고서 등 공개 정보 검색 요구

#### 태스크 2: 재무 및 밸류에이션 분석
- subject: `{회사명} 재무 데이터, 수익성 및 밸류에이션 분석`
- description 포함:
  1. 최근 3-5년 매출, 순이익, 영업이익 추이
  2. 수익성 지표: ROE, ROA, 매출총이익률, 영업이익률
  3. 현금흐름 분석: 영업 현금흐름, 잉여현금흐름, 자본지출
  4. 재무상태표 건전성: 현금 보유, 부채 비율, 유동성
  5. 밸류에이션 분석: PE/PS/PB/EV 등, 역사 및 동종 업계 비교
  6. 안전마진 평가: 내재 가치 vs 현재 주가
  7. **금융 엄밀성 검증 (반드시 Bash로 도구 호출, 암산 금지)**:
     - 시가총액 검산: `python3 ~/ai-berkshire/tools/financial_rigor.py verify-market-cap --price {가격} --shares {주식수} --reported {보고서 시총} --currency {통화}`
     - 밸류에이션 검산: `python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation --price {가격} --eps {EPS} --bvps {주당순자산}`
     - 핵심 데이터 교차 검증: `python3 ~/ai-berkshire/tools/financial_rigor.py cross-validate --field {필드} --values '{JSON}' --unit {단위}`
     - 3시나리오 밸류에이션: `python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario --price {가격} --eps {EPS} --shares {주식수억} --growth {낙관} {중립} {비관} --pe {낙관PE} {중립PE} {비관PE}`
     - 도구 출력 결과를 보고서에 검증 기록으로 직접 삽입

#### 태스크 3: 업계 및 경쟁 분석
- subject: `{업종} 업계 구도 및 {회사명} 경쟁 태세 분석`
- description 포함:
  1. 업계 규모와 성장: 시장 규모, 성장률, 침투율
  2. 경쟁 구도: 주요 경쟁사 시장점유율, 경쟁 전략 비교
  3. 핵심 경쟁자 위협 평가: 주요 경쟁사 개별 분석
  4. 각 세부 트랙 구도
  5. 업계 트렌드: 기술 변화, 정책 영향, 신규 진입자
  6. 가치사슬 분석: 상중하류 가치 배분
  7. 최신 업계 데이터와 경쟁 동향 검색 요구

#### 태스크 4: 리스크 및 경영진 평가
- subject: `{회사명} 투자 리스크 및 경영진 품질 평가`
- description 포함:
  1. 경영진 평가: CEO 능력권, 정직성, 전략적 안목, 자본 배분 능력, 과거 의사결정 품질
  2. 규제 리스크: 현재 및 잠재적 규제 영향
  3. 경쟁 리스크: 각 경쟁사 위협 수준 평가
  4. 사업 리스크: 신사업 적자, 확장 불확실성
  5. 거시 리스크: 경제 사이클, 업계 사이클 영향
  6. 지배구조: 지분 구조, 관련자 거래, 주주 환원 정책
  7. 장기 확실성: 10년 후 회사는 어떻게 될까? 무엇이 사업 모델을 전복할 수 있을까?
  8. 최신 규제 동향, 경영진 발언 등 검색 요구

### 4단계: 4개 병렬 Agent 시작

Task 도구로 4개 Agent를 **동시에** 시작합니다 (**반드시 같은 메시지에서 병렬 호출**).

각 Agent 설정:
- `subagent_type`: `general-purpose`
- `run_in_background`: `true`
- `team_name`: 해당 팀명
- `name`: 해당 역할명 (business-analyst / financial-analyst / industry-researcher / risk-assessor)

각 Agent의 prompt 템플릿:

```
당신은 {회사명} 투 research 팀의 "{역할 한글명}"이며, {대가명} 투자 관점에서 {회사명}을 분석하는 책임을 맡고 있습니다.

태스크 #{태스크번호}를 완료하세요: {태스크 subject}

구체적 요구사항:
{태스크 description 내용}

**연구 방법**:
- WebSearch를 사용해 최신 공개 정보 검색 (재무제표, 업계 보고서, 뉴스)
- **재무 데이터는 반드시 두 개의 독립 출처**에서 가져와야 하며, `skills/financial-data.md` 규범에 따라 실행 (미국주: macrotrends+stockanalysis; 홍콩주: aastocks+macrotrends; A주: 동방재부+거조자신), 두 출처 오차 >1% 시 반드시 표시
- 데이터 정확성을 보장하고, 핵심 데이터에 출처 표기
- 분석은 표면에 그치지 말고 깊이 있게 수행

**출력 요구사항**:
- 보고서는 상세하게 작성하고, 핵심 데이터는 Markdown 표로 제시
- 각 분석 차원마다 명확한 결론과 평점 부여
- 보고서 마지막에는 해당 차원의 전체 결론 포함

**완료 후**:
1. TaskUpdate를 사용해 태스크 #{태스크번호}를 completed로 표시
2. SendMessage를 통해 완성된 분석 보고서를 team-lead에게 전송 (type: "message", recipient: "team-lead")
```

### 5단계: 보고서 수신 및 진행 추적

- 사용자에게 실시간 진행표 표시 (어느 Agent가 완료되었고, 어느 것이 연구 중인지)
- 보고서를 받을 때마다 진행 상황 업데이트 및 해당 보고서의 핵심 포인트 (3-5개) 제시
- 4개 보고서가 모두 도착할 때까지 대기

### 6단계: 팀 멤버 종료

모든 보고서를 받은 후, 4개 Agent에게 shutdown_request 전송 (SendMessage 사용, type: "shutdown_request").

### 7단계: 최종 보고서 종합

4개 분석 보고서를 종합해 다음 구조의 최종 보고서 출력:

---

#### 1. 한 줄 결론
> 한 문단 (50-100자)으로 투자 가치 여부와 핵심 논리를 요약

#### 2. 4차원 평점 종합표
| 차원 | 프레임워크 | 평점(1-5★) | 핵심 판단 |
|------|------------|------------|-----------|

종합 평점: X / 5

#### 3. 핵심 데이터 요약
주요 재무 및 영업 지표 표 (최근 2년 비교)

#### 4. 각 차원 분석 요약
각 차원에서 가장 중요한 발견 3-5개 추출

#### 5. 투자 제안
| 전략 | 제안 | 가격 구간 |
|------|------|-----------|
| 공격형 | ... | ... |
| 안정형 | ... | ... |
| 보수형 | ... | ... |

#### 6. 주요 리스크
3-5개의 핵심 리스크 + 완화 요인

#### 7. 결론
최종 판단 + "거울 테스트" 통과/미통과 여부

---

전체 보고서를 `reports/{회사명}/최종보고서.md` 및 기타 하위 파일로 작성.
