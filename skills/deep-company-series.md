# 심층 회사 시리즈: 8편 장문으로 한 회사를 해부하다

$ARGUMENTS 에 대해 8편의 심층 장문 시리즈를 작성합니다. 이는 공중계/비디오 채널 등 공개 채널에 게시하기 위한 것입니다. **핵심 IP는 "잘 쓰는 것"이 아니라 "잘 고치는 것"입니다 — 99%의 재무 기사가 이 skill의 사실 검증 기준을 위반하고 있습니다.**

참고 샘플: `reports/腾讯/《看懂腾讯》/`

---

## 1. 트리거 시나리오

사용자가 한 회사에 대해 "교과서 수준"의 심층 연구를 원하고, **시리즈 장문** 형태로 공개 게시하기를 원합니다. 단편 리서치 보고서와의 차이:
- 8편 약 12만 자, 인식 재정립부터 의사결정 프레임워크까지 완전한 클로징
- 각 편은 독립적으로 완성된 글(단편 공유에 적합)이지만, 하나의 밸류에이션/경영진/가격 판단 체계를 관통
- "90분을 투자해 한 회사를 이해하려는" 독자를 위한 글, 증권사 고객을 위한 글이 아님

**이 skill을 사용하지 말아야 할 상황**: 단편 리서치 보고서, 분기 실적 코멘트, 업계 연구 — 이러한 경우 `/investment-research`, `/earnings-review`, `/industry-research`를 사용하십시오.

## 2. 시리즈 편목 템플릿 (8편)

| # | 편명 템플릿 | 핵심 질문 | 분량 |
|---|-------------|-----------|------|
| 01 | 당신이 X를 이해했다고 생각하지만, 실제로는 아니다 | 인식 재정립: 3가지 흔한 오해 깨기 | 4,000-5,000 |
| 02 | X의 해자 — `<비즈니스 본질 한 줄>` | 해자가 깊은가, 향후 5/10년에도 존재할 것인가 | 6,000-8,000 |
| 03 | X의 최대 이익 엔진 — `<가장 수익성 높은 사업>` | 주력 사업은 무엇이며, 왜 지속 가능한가 | 6,000-8,000 |
| 04 | X가 장부에 숨겨둔 또 하나의 회사 — `<숨겨진 자산>` | 투자 포트폴리오 / 자회사 / 숨겨진 가치 | 8,000-10,000 |
| 05 | AI(또는 현재 서사) 시대에 X는 승자인가 패자인가 | 시대 변수: 사업부별로 AI 영향을 분해 | 8,000-10,000 |
| 06 | 버핏 방식으로 X의 재무제표를 해부하다 | 재무 심층: 매출총이익률/FCF/ROE/SBC | 8,000-10,000 |
| 07 | `<경영진 금언>` — X의 경영진은 맡길 만한가 | 자본 배분 규율 + 정직성 검증 + 후계자 | 8,000-10,000 |
| 08 | 얼마에 사는 것이 가치 있는가, 어떤 시그널에서 반드시 팔아야 하는가 (시리즈 종장) | DCF 3시나리오 + 레드라인 목록 + 비중 프레임워크 | 10,000-12,000 |

추가로 `00-시리즈 설명.md` 를 목차 인덱스로 작성하며, 이는 게시하지 않습니다.

## 3. 작성 스타일 규범

### 어조
- **직설적, 날카로움, 군더더기 없음** — 첫 문장에서 숫자 또는 반상식 결론 제시
- **가치 투자 프레임워크** — 버핏/멍거/단용평/리루 관점이 자연스럽게 스며들게 (명언을 나열하지 않음)
- **선입견 없음** — 먼저 데이터를 제시하고, 논리를 추론한 후, 결론 도출
- **양면 제시** — 모든 핵심 판단에 "그러나 다른 한편으로는..." 반대 의견 첨부
- **공중계 체감** — 앞 18-20자는 독립적으로 설 수 있어야 함 (모바일 미리보기)

### 금지어

| 금지 | 이유 | 대체 |
|------|------|------|
| 분명히 / 필연적으로 / 반드시 | 주관적 절대화 | 데이터에 따르면 / 증거가 보여준다 |
| 나는 생각한다 / 나는 느낀다 | 주관적 어조 | 삭제하거나 "본 프레임워크에 따르면"으로 변경 |
| 교과서 수준 / 신의 한 수 | 트래픽용 과장 | 구체적 사실 서술 |
| 심각하게 불일치 / 심각하게 저평가 | 강한 주관 단어 | 구체적 할인 비율 제시 |
| 완벽한 / 흠잡을 데 없는 | 일방적 판단 | 반대 관찰 추가 |

### 제목 스타일

- **반전 숫자** 또는 **반컨센서스 결론**을 후크로 사용 ("15년 7차례 도전 모두 실패", "연봉 4,292만 위안이 이익의 0.0017% 차지")
- 부제목은 중립적이고 내용을 요약 ("—— `<본질 판단>`")
- **트래픽용 비유 피하기**: "작은 버핏", "중국판 X", "YYDS" 등 일절 사용 금지
- 전문 독자가 익숙한 용어 사용 ("버크셔"가 아닌 "버핏", 회사명이 인명보다 우선)

## 4. 엄격한 사실 검증 체크리스트 (핵심 IP)

### 쓰기 전에 경계해야 할 "가짜 정밀" 함정

1. **확률 가중 기대값**: `30% × A + 50% × B + 20% × C = 기대 +X%` 같은 계산은 거의 모두 쓰레기 — 확률 배정은 순전히 주관적이며, 독자에게 잘못된 정밀감을 준다. **시나리오 + 트리거 조건 + 방향만 나열하고, 가중 기대값은 계산하지 말 것**.
2. **제3자 추정 MAU/점유율**: QuestMobile/七麦/CBNData 등 산정 기준 차이가 매우 크다 (동일 시점에도 2-3배 차이 날 수 있음). **가장 신뢰할 수 있는 두 가지를 anchor로 사용하고, 나머지는 정성적 서술로**.
3. **역사적 성장률 선형 외삽**: `2025년 +33% × 5년 복리 → 2030년 X` 는 금융 문맹식 예측이다. **시나리오 가정 + 상/하 구간 + 약속이 아님**.
4. **미공개 지분 비율**: 바이트댄스, Halti 등 비상장사 지분은 **공개된 적이 없음**. **구간을 주고, "알 수 없음"이라고 표기**.
5. **강한 귀인**: 경쟁사 실패 = X 때문. 다중 원인을 모두 나열하고, **본문에서 단일 귀인은 하지 말 것**.

### 수정 시 반드시 실행할 7가지 검사

```
□ 1. 편 간 숫자 일관성: 총 시가총액, Non-IFRS 순이익, 핵심 지분 % 전 시리즈 정렬
□ 2. 산정 기준 표기: Non-IFRS / GAAP / Non-IFRS-SBC / FCF 각각 어떤 것을 사용했는지 본문에 명확히
□ 3. 중복 가산 스캔: 이미 연결된 자회사가 "투자 포트폴리오"에 다시 들어가지 않도록, SOTP에서 이중 계산 금지
□ 4. 횡단 비교 공정성: "주력 사업 PE(현금+포트폴리오 제외)" vs "경쟁사 PE(제외 안 함)" 비교 금지
□ 5. 확률 가중 전부 삭제: 위 1번 참조
□ 6. 절대화 표현 전부 완화: grep "분명히|필연|심각|교과서|완벽"
□ 7. 제3자 데이터 출처 표기: 재무제표가 아닌 모든 데이터 뒤에 "(출처: X)" 추가
```

### 모델 선호

쓰기 전에 **알려진 하드 에러 리스크를 먼저 나열**:
- 역사적 수익 배수: 누적 투입 기준으로 사용 (예: Riot 33배이지 58배가 아님)
- 지분 비율: 최신 Futu/재무제표 기준 사용 (예: 텐센트가 메이퇀 보유 1.5%이지 6.4%가 아님)
- "배당 분배" 회계 처리: IFRIC 17에 따라 선고일에 처분 이익으로 간주 (예: 징동 2021, 메이퇀 2022지만 금액은 작음)
- 총 발행 주식수 반등: SBC 집중 연초 부여로 주식수가 단기 상승할 수 있음

## 5. 실행 흐름

### 단계 1: 조사 (01-02편 작성 전에 완료)

1. 회사 최근 5년 연례 보고서, 최신 분기 보고서 읽기
2. 최소 3부의 독립적 매도 측 리서치 보고서 읽기 (컨센서스 + 반컨센서스 찾기)
3. `/investment-team` 또는 `/investment-research` 로 먼저 내부 연구 초안 생성
4. 사용자와 8편의 핵심 논점 확인 (다 쓴 후 방향이 틀렸음을 발견하는 일 방지)

### 단계 2: 작성 (01→08 순서대로 쓰며, 건너뛰지 말 것)

- 각 편 작성 후 `reports/{회사명}/《看懂{회사명}》/0X-XX.md` 에 저장
- 즉시 GitHub에 푸시하지 말고 — 사용자가 검토할 때까지 대기
- 사용자가 수정 의견을 주면 수정
- 수정 완료 후 git push

### 단계 3: 편 간 일관성 스캔 (08편 전부 작성 후)

Explore agent를 병렬로 배치하여 8편에 대해 다음 검사를 실행:
1. 동일 숫자(시가총액, 순이익, 지분 비율)가 편 간 일관적인가
2. 동일 용어(FBS, SBC, Non-IFRS)가 처음 등장할 때 설명이 되어 있는가
3. 인용 관계: 02편에서 "06편 참조"라고 한 것이 실제로 일치하는가
4. 요점 요약 vs 본문의 숫자가 일치하는가

### 단계 4: 게시 전 최종 검사

```bash
# 푸시 전에 반드시 로컬에서 grep 1회 실행 (ai-berkshire 프라이버시 규칙에 따름)
grep -r "linxuan\|/Users/\|<사용자 회사 꽃말>" reports/ | head
```

오류가 없음을 확인한 후 `git pull --rebase && git commit && git push` 를 실행합니다.

## 6. 수정 의견 처리 흐름

사용자가 수정 의견을 줄 때 다음 순서로 처리:

### 1. 먼저 사실을 검증 (바로 수정하지 말 것)

사용자가 "X 데이터가 틀렸다"고 하면, 먼저 Bash/Read로 원본 데이터를 교차 검증:
- ai-berkshire 프로젝트 내 동일 회사의 earnings/재무 보고서 확인
- Futu/공식 공시 확인
- "사용자가 말한 데이터 vs 내가 조사한 데이터 vs 내가 이전에 사용한 데이터" 3자 비교 제시

### 2. 수정 수준 판단

| 수준 | 유형 | 처리 |
|------|------|------|
| 🔥 하드 에러 | 숫자 오류, 귀인 오류, 산정 기준 오류 | 반드시 수정, 주저하지 말 것 |
| ⚠️ 주관화 | 강한 주관 단어, 절대화, 트래픽용 비유 | 완화하거나 삭제 |
| 🔬 세밀도 | 출처 표기, 산정 기준 세분화 | 우선순위 낮음, 가독성과 균형 |
| ❓ 신뢰 불가 | 제3자 추정 차이가 매우 큼 | **수정보다 삭제가 더 안전** (사용자가 명시적으로 지시한 경우) |

### 3. 수정 후 연동 검사

한 곳을 수정하면 "이 숫자/개념을 인용하는 다른 곳은 어디인가"를 먼저 생각. 예:
- 시가총액을 수정 → 전 시리즈 PE / 주력 사업 PE / 할인율 / FCF Yield 연동 수정
- 지분 %를 수정 → TOP 10 순위 + 역사적 지분 표 + 매도 목록 연동 수정
- 용어 산정 기준을 수정 → 최초 정의 + 이후 인용 + 요점 요약 연동 수정

### 4. 푸시 후 즉시 보고

```
푸시 성공 (commit hash).
[N] 건 수정 요약 [표 포함]:
- 무엇을 수정했는가
- 연동하여 무엇을 수정했는가
- 아직 수정하지 않은 것은 무엇인가

다음 단계 지시 대기.
```

## 7. 본 skill 이 하지 않는 것

- **독자를 대신해 투자 의사결정을 하지 않음** — 모든 편 말미에 "투자 조언이 아님" 명시
- **주가 예측을 하지 않음** — "시나리오 + 트리거 조건"만 제시
- **"기대 연환산 수익" 가중값을 계산하지 않음** — 주관적 확률 배정은 독자를 오도함
- **"X 대가도 보유하고 있다"를 쓰지 않음** — 다른 사람의 보유로 자신의 판단을 뒷받침하는 것은 가치 투자에 반함
- **8편을 모두 쓰도록 강요하지 않음** — 특정 편에 충분한 독립적 내용이 없다면 (예: 어떤 회사의 경영진이 특별하지 않다면) 다른 편에 병합하거나 편 수를 줄임

## 8. 준수 및 프라이버시

- 모든 공개 보고서는 **공개 정보만** 사용 (재무제표, 공식 공시, 증권사 리서치, 유명 제3자 기관)
- **사용자 개인정보** (회사 꽃말, 내부 IM, 미공개 보유 정보)는 사용하지 않음
- 푸시 전에 반드시 grep으로 `linxuan` / `/Users/` / 사용자 회사 꽃말 등 프라이버시 필드를 스캔 ( `~/.claude/projects/-Users-linxuan/memory/feedback_privacy_upload.md` 참조)
- 공개 서명은 사용자의 다층 정체성 전략에 따르며, 혼용하지 않음

## 한 줄 요약

**《X를 보다 시리즈》를 쓰는 핵심 역량 ≠ 잘 쓰는 것, 엄격하게 고치는 것이다** —
89%의 재무 장문은 가짜 정밀 숫자, 주관적 가중 기대값, 절대화 표현 때문에 죽는다. 이 skill의 존재 이유는 이러한 함정을 모두 표시하여, 쓰기 전에 피하고, 쓴 후에 깨끗이 청소하는 것이다.
