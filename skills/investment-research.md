# 투자 연구：버핏-멍거-단용평-리루 4대 거장 종합 분석 프레임워크

$ARGUMENTS 에 대해 체계적인 투자 연구 분석을 수행합니다.

## 연구 프레임워크

버핏, 멍거, 단용평, 리루 네 명의 투자 대가의 방법론에 기반하여, 아래 일곱 모듈 순서대로 연구를 실행합니다:

### 전제 단계: AI 연구 편향 자각 (반드시 실행)

연구를 시작하기 전에 해당 회사의 "AI 연구 가능성"을 평가하고 잠재적 데이터 편향을 식별합니다:

**정보 풍부도 등급**:
| 등급 | 특징 | AI 연구 함정 | 대응 전략 |
|------|------|-------------|----------|
| A급 (정보 충족) | 상장 다년, 증권사 커버리지 많음, 미디어 보도 밀집 | 컨센서스 과도, AI 출력이 시장 가격에 수렴, 알파 제한적 | 반대 검증 집중: 똑똑한 사람들이 왜 사지 않는가? 간과된 리스크는 무엇인가? |
| B급 (정보 보통) | 상장 1-3년, 커버리지 제한적, 일부 데이터 추정 필요 | AI가 "합리적 추정"으로 공백을 메우려 함, 완전해 보이지만 실제로는 거짓 확실성 | 모든 추정 데이터에 신뢰도 표기, "근거 있는 추정"과 "근거 없는 채움" 구분 |
| C급 (정보 희박) | 갓 상장/비인기주/신흥시장, 커버리지 거의 없음 | AI가 자료 부족으로 과도하게 보수적 판단, "알 수 없음=좋지 않음" 오판 | 제1원칙 질문(아래 참조) 사용, 제한된 정보에서 비즈니스 본질 추출 |

**C급 회사의 제1원칙 연구법**:
공개 자료가 부족할 때 "완전해 보이는" 보고서를 억지로 만들려 하지 말고, 다음 근본 질문에 집중합니다:
1. 고객은 누구인가? 왜 돈을 내는가? 대안이 있는가?
2. 재구매를 이끄는 동력은 무엇인가? 습관, 락인, 아니면 지속적인 신가치 창출인가?
3. 경쟁사가 100억을 투자해 이 사업을 복제할 수 있는가?
4. 경영진이 어떤 주요 의사결정을 했는가? 그 결정들은 어떤 판단력과 가치를 반영하는가?

**편향 자가 점검 목록** (연구 전 과정에서 경계 유지):
- [ ] 나의 "확실성" 느낌은 비즈니스 본질에서 오는가, 아니면 자료의 양에서 오는가?
- [ ] 이 회사의 자료량을 절반으로 줄이면 결론이 바뀌는가?
- [ ] AI가 출력한 분석이 시장 컨센서스와 매우 유사한가? 그렇다면 나의 정보 우위는 어디에 있는가?
- [ ] "공개 자료는 적지만 비즈니스 본질이 극히 우수할" 가능성이 과소평가되고 있지는 않은가?

정보 풍부도 등급 결과를 보고서 서두에 기록하고, 최종 결론에서 "AI 연구 신뢰도"와 "실제 투자 확실성"의 차이를 명시합니다.

### 첫 번째 단계: 데이터 수집

> **데이터 소스 규범**: `skills/financial-data.md` 참조. 모든 재무 데이터는 두 개의 독립 출처에서 확보해야 하며, 오차 >1% 시 반드시 표시.
> - 미국 주: macrotrends (주) + stockanalysis (부)
> - 홍콩 주: aastocks (주) + macrotrends ADR (부)
> - A주: 동방재부 (주) + 거조자신 (부)

Task 도구를 사용해 백그라운드 Agent를 시작하고, 네트워크에서 다음 데이터를 수집합니다:

1. 수익 구조: 최근 회계연도 및 최근 4분기 사업부별 매출, 성장률, 매출총이익률
2. 재무 지표: 최근 5년 매출, 순이익, 매출총이익률, 영업이익률, 잉여현금흐름, 현금 보유
3. 경쟁 구도: 시장 점유율, 주요 경쟁사 대비
4. 비즈니스 모델과 해자: 핵심 경쟁우위 원천
5. 기술 역량: 핵심 기술 스택, R&D 투자
6. 경영진: 창업자/CEO 이력, 지분 비율, 주요 의사결정 기록
7. 업계 전망: TAM(총 이용 가능 시장), 성장 예측
8. 리스크 요인: 지정학, 규제, 공급망 등
9. 현재 밸류에이션: 시가총액, PE, PS, PEG, EV/Revenue
10. 공매도/매수 양측 핵심 논점

#### 데이터 교차 검증 (반드시 실행, 금융 엄밀성 도구 사용)

데이터 수집 완료 후, **반드시 `tools/financial_rigor.py`를 호출하여 핵심 데이터를 프로그램적으로 검증**하고 LLM 암산 오차를 제거합니다.

**반드시 검증해야 할 데이터 포인트**:
- 총 발행 주식수 (거래소, Yahoo Finance, StockAnalysis 등 최소 2개 출처 확인)
- 현재 주가와 시가총액 (**주가 × 총 발행 주식수를 수작업 계산하여 보고서 시가총액과 비교, 단위 오류 방지**)
- 최근 회계연도 매출과 순이익 (회사 연례 보고서 + 최소 1개 제3자 출처 확인)
- 현금 보유와 순현금 (현금 + 단기투자 - 총부채, 산정 기준 차이 주의)
- 경영진 지분 비율 (경제적 권리와 의결권 구분, AB주 구조 주의)

**강제 검증 단계 (Bash로 도구 호출)**:

Step 1 — 시가총액 검산 (정확한 십진수, 부동소수점 아님):
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-market-cap \
  --price {주가} --shares {총 발행 주식수} --reported {보고서 시가총액} --currency {통화}
```

Step 2 — 핵심 데이터 다중 출처 교차 검증:
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py cross-validate \
  --field {필드명} --values '{"출처1": 수치, "출처2": 수치}' --unit {단위}
```
매출, 순이익, 현금 보유 각각에 대해 실행.

Step 3 — 밸류에이션 지표 정확 검산 (PE/PB/ROE/FCF Yield 등):
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation \
  --price {주가} --eps {EPS} --bvps {주당 순자산} --fcf-per-share {주당 FCF} --dividend {주당 배당}
```

**검증 규칙**:
1. 모든 핵심 데이터 포인트는 최소 2개의 독립 출처
2. 출처 간 차이가 발견되면 회사 연례 보고서/거래소 데이터를 우선 사용하고 차이 원인 명시
3. **모든 계산 관련 데이터는 반드시 도구로 검산해야 하며, LLM 암산 금지**
4. 도구 출력 결과를 보고서 부록 "핵심 데이터 교차 검증 기록"에 직접 삽입
5. 도구가 ❌ 과도한 편차를 보고하면 원인을 조사한 후에야 분석 계속

**일반적인 오류 방지**:
- 시가총액 단위: 홍콩달러 억 vs 위안 억 vs 달러 억 — 0 하나를 빼먹거나 추가하기 쉬움
- FCF 산정 기준: 자본적 지출 정의가 출처마다 다를 수 있음 (리스, 인수 포함 여부 등)
- 부채 산정 기준: 운용 리스 부채 포함 여부
- 지분 비율: AB주 회사의 경제적 권리 ≠ 의결권

### 두 번째 단계: 비즈니스 본질 분석 — 단용평 "옳은 사업"

분석 포인트:
- 한 문장으로 이 사업의 본질 정의
- 수익 구조 분해 (표/차트)
- 5년 수익성 추이 (표/차트)
- 비즈니스 모델 캔버스: 일회성 판매 vs 구독/재구매? 하드웨어 vs 소프트웨어 vs 플랫폼?
- 생태계 점착성/고객 락인 강도
- 매출총이익률 수준과 동종 업계 비교, 왜 높/낮은지 설명
- 영업 레버리지 분석
- **단용평식 추궁**: 이 사업이 좋은 점은 무엇인가? 한 문장으로만 설명한다면 무엇인가?

### 세 번째 단계: 해자 평가 — 버핏 "경제적 해자"

다섯 가지 해자 유형을 하나씩 검증:

| 해자 유형 | 검증 방법 |
|-----------|-----------|
| 브랜드/가격 결정력 | 판매량 손실 없이 가격을 인상할 수 있는가? |
| 전환 비용 | 고객이 경쟁사로 이동하는 비용이 얼마나 높은가? |
| 네트워크 효과 | 사용자가 많을수록 제품이 더 좋아지는가? |
| 규모 효과 | 규모가 가져오는 비용 우위가 얼마나 큰가? |
| 기술/특허 장벽 | 기술 우위가 몇 년인가? 복제될 수 있는가? |

해자 추이 분석: 과거 5년 동안 넓어졌는가 좁아졌는가? 향후 5년 전망.

**버핏식 추궁**: 10년 후에도 이 해자가 남아 있을까? 무엇이 그것을 파괴할 수 있는가?

### 네 번째 단계: 역발상 사고와 리스크 목록 — 멍거 "반대로 생각하기"

- "이 회사가 실패할 수 있는 모든 경로" 나열 (표: 경로/확률/영향도)
- 역사적 유추: 유사한 위치에 있었던 역사적 회사와 그 결말은?
- 학제 간 분석: 네트워크 효과 이론, 기술 채택 곡선, 경쟁 게임 이론 등으로 교차 검증
- 편향 자가 점검: 서사 편향, 닻내림 효과, 생존자 편향
- 공매도 측 핵심 논점 수집

**멍거식 추궁**: 내가 가장 크게 실수할 가능성이 있는 곳은 어디인가? 똑똑한 사람들이 왜 사지 않거나 공매도하는가?

### 다섯 번째 단계: 경영진 평가 — 단용평 "옳은 사람" + 버핏 "경영진 정직성"

- CEO/창업자 주요 의사결정 되짚기 (표: 시점/결정/결과/평점)
- 자본 배분 능력: R&D 투자수익률, M&A 성공률, 자사주 매입 타이밍
- 주주 이익 일치성: 경영진 지분 보유, 보수 구조, 매도 기록
- 조직 역량: 팀 안정성, 핵심 인재 리스크
- 기업 문화 특징

**단용평식 추궁**: CEO가 은퇴하면 이 회사는 여전히 경쟁력을 유지할 수 있는가?

### 여섯 번째 단계: 업계와 문명 트렌드 — 리루 "문명 진화 프레임워크"

- 해당 업계가 "문명급 패러다임 전환" 단계에 있는지 판단
- 역사적 기술 혁명 유추 (증기기관/전기/인터넷/AI)
- TAM 성장 곡선과 천장 분석
- 산업 가치사슬 내 회사 위치
- 기술 경로 리스크
- 고객/공급자 집중도 분석

**리루식 추궁**: 20년 후를 돌아보면, 이 회사는 "이 시대의 스탠다드 오일"인가, 아니면 "잠깐 빛났던 3Com"인가?

### 일곱 번째 단계: 밸류에이션과 안전마진 — 버핏 "내재 가치" + 단용평 "옳은 가격"

- 현재 시장 가격 (핵심 밸류에이션 지표 표) — **반드시 도구로 검산**
- 역DCF: 현재 주가가 어떤 성장 기대를 내포하고 있는가?
- 3시나리오 밸류에이션 — **반드시 도구로 정확 계산, 암산 금지**:
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario \
  --price {주가} --eps {EPS} --shares {총 발행 주식수 억} \
  --growth {낙관성장률} {중립성장률} {비관성장률} \
  --pe {낙관PE} {중립PE} {비관PE} --years 3 --currency {통화}
```
- 자체 역사적 밸류에이션 대비
- 동종 업계 밸류에이션 대비

**단용평식 추궁**: 만약 증시가 내일부터 5년간 폐쇄된다면, 이 가격에 보유할 용의가 있는가?

## 출력 요구사항

1. 모든 분석은 데이터로 뒷받침되어야 하며, 데이터 출처를 첨부
2. Markdown 표를 사용하여 핵심 데이터를 제시
3. 각 모듈 끝에는 해당 대가의 "추궁"이 반드시 포함되어야 함
4. 최종적으로 전체 보고서를 `~/[회사명]투자연구보고서.md` 에 작성
5. 결론은 명확해야 하며, 매수/관망/회피 제안을 회피하지 말 것
6. 밸류에이션 부분에서는 구체적인 가격 구간을 제시
7. **보고서 서두**에는 반드시 "정보 풍부도 등급"(A/B/C)과 "AI 연구 한계 선언" 포함
8. **보고서 말미**에는 "AI 분석 신뢰도"와 "투자 확실성"을 구분 — 전자는 자료량에, 후자는 비즈니스 본질에 달려 있음. 독자에게 본 보고서의 어떤 결론이 충분한 데이터에 기반하고 어떤 결론이 제한된 정보의 추론인지 명확히 알릴 것
9. 회사가 C급(정보 희박)에 해당한다면, 보고서 말미에 "1차 검증이 필요한 질문 목록"을 반드시 나열 — 독자가 현장 조사, 제품 체험, 공급망 인터뷰 등을 통해 AI의 사각지대를 보완하도록 권고

## 데이터 추출 검사 (준출 프로세스)

보고서를 파일로 작성한 후, **반드시** 데이터 추출 검사를 실행하고 통과해야 게시할 수 있습니다:

**Step 1 — 추출 검사 목록 (15% 무작위 샘플링):**
```bash
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report <보고서 파일 경로>
```
JSON 템플릿을 출력하며, 각 항목은 `fetched_value` (채울 값)를 포함합니다.

**Step 2 — 데이터 취득 및 확인:**
목록의 각 데이터 포인트에 대해 `skills/financial-data.md` 규범에 따라 신뢰할 수 있는 출처에서 데이터를 취득
(미국 주: macrotrends+stockanalysis; 홍콩 주: aastocks+macrotrends; A주: 동방재부+거조자신),
`fetched_value` / `fetched_source` / `fetched_value2` / `fetched_source2` 에 채웁니다.

**Step 3 — 판정 출력:**
```bash
python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<채워진 JSON>' \
  --report <보고서 파일명>
```

- **【준출】**: 모든 검사 포인트 편차 ≤ 1% → 보고서 게시 가능
- **【반려】**: 임의의 포인트 편차 > 1% → 해당 데이터 수정 후 재검사, 준출될 때까지 반복
